## Features

- Dialog-based wallet picker listing detected wallet adapters in the browser.
- Dropdown menu with truncated public key and disconnect action once connected.
- Loading state on the trigger button while a connection is in progress.
- Configurable Solana network and auto-connect behavior.

## API Reference

### WalletAdapter

This component takes no props. State is managed internally via the `useWallet` hook — connection status, the active public key, and the list of detected adapters all come from context.

### WalletContextProvider

Wraps your application with Solana connection and wallet context.

| Prop       | Type        | Default     | Description                   |
| ---------- | ----------- | ----------- | ----------------------------- |
| `network`  | `Network`   | `"testnet"` | Solana cluster to connect to. |
| `children` | `ReactNode` | —           | Application content.          |

## Notes

- Renders a `DialogTrigger` button when no wallet is connected, opening a list of detected wallet adapters to choose from.
- Once connected, swaps to a `DropdownMenu` showing the truncated public key with a disconnect action.
- Wallet selection errors are currently only logged to the console — consider surfacing these via toast in production.
- The `supportedWallets` filter is defined in the component but not currently applied; the dialog lists all detected adapters.
